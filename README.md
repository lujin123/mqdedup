# mqdedup

消息队列中消息去重的一种方式，作为一个练手工具，写着玩玩

实现参考[这个](https://jaskey.github.io/blog/2020/06/08/rocketmq-message-dedup/)

这个方案并不是完美的，存在的问题有：

1. 使用 mysql 或者 redis 作为去重存储，可能会成为性能瓶颈
2. 对于无法回滚的业务，出错后还是会重新发送消息，导致接受到重复消息
3. 业务成功后，但是对消息状态标记为成功有可能会失败，这会也会导致在标记失效后，消息重复投递

总体来说，可以处理绝大部分正常场景，对于异常场景，需要加强监控

对于严格要求幂等的业务场景，需要自行在数据库中做幂等处理，例如 版本号和状态机
